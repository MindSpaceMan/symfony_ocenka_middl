### db: orm, mysql
### linter - psalm
### tests - phpunit
### запуск:
```
composer install
установить подключение к своей версии mysql 
через смену данных в .env файле
symfony d:d:c
symfony d:m:m
запустить 1 раз:
php bin/console doctrine:fixtures:load --append
symfony serve
``` 

p.s.: docker, nginx и любое дополнительное окружение не ставил,
так как этого не было явно указано в задании. 
```
Также я не менял архитектуру приложения: например DDD. Так как понимаю - 
что тестовое задание действительно интересный этап,
но - настолько сразу поднимать архитектуру DDD - я 
предпочел сфокусироваться на хорошем выполнении поставленной задачи
И внедрению нестандартного подхода, понимая, что 
в реальном проекте (коммерческом) можно будет и архитектуру,
при потребности от руководства изменить в 
нормальном, качественном темпе.
```
